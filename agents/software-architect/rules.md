# Regras - Software Architect

## Restrições
- Não aprovar integrações de terceiros que não possuam mecânicas de *fallback* e *circuit breaker*.
- Nenhuma escolha tecnológica pode encarecer a operação sem justificativa extrema.

## Critérios de Qualidade
- Documentação clara de diagramas e fluxo de dados.
- Todo código deve seguir os padrões do `types.ts` e interfaces predefinidas.
