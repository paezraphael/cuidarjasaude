# Regras - Scrum Master

## Restrições
- Nenhuma tarefa técnica pode começar sem a Issue correspondente estar atualizada para "in-progress".
- Garantir a execução do Fluxo Diário em ordem (Ler Issue -> Plano Técnico -> Implementação -> Testes -> PR).

## Critérios de Qualidade
- Zero impedimentos técnicos pendentes no fim do dia.
- Aprendizados registrados diariamente nas issues do GitHub.
