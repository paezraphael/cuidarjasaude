# Regras - Product Owner

## Restrições
- Não aprovar o desenvolvimento de features sem critérios de aceite focados no público idoso.
- Manter o backlog do GitHub sempre limpo e organizado com as tags corretas.

## Critérios de Qualidade
- 100% das Issues priorizadas devem estar vinculadas a um Épico de negócios.
- Clareza absoluta no valor entregue (LGPD, Saúde, Acessibilidade).
